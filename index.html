とりあえず、Generatorを使って作ってみたよ

GitHub と、 GitHub Pages のことをちょっと調べています。 とりあえず、今は　GitHub のサイトで、 Generator を使って Markdown で書いてみています。

* Generator + Markdown はけっこう簡単に作れていいね
* デザインテンプレートもたくさんあるから、プログラマの人は助かるね
* 自由度はあまりないかな？

これから他のプロジェクトでページを作って、誰かと一緒に管理することを考えると、 Generator じゃないほうがいいのかな。 テンプレートをうまく使い回すのも手かな。

GitHub for Mac で Sync しようか迷っているよ

最近 Git の GUI は SourceTree にほぼ移行してしまったので、 できれば SourceTree でやりたいと思っています。 ふつうに HTML を作って Sync させる分には問題ないよね。

Generator を使わない方法もあるんだね

ページジェネレータを使いたくない場合は、"gh-pages"という名前の branch を push すればいいのかな。 Jekyll をサポートしているから、それでブログ的にも使えるのかな。ふむふむ。
